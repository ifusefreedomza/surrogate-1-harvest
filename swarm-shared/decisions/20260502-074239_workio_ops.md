#### Step 1: Update `train.py` to Use HF CDN Bypass for Dataset Download
Modify the `train.py` script to download dataset files from the HF CDN using the `https://huggingface.co/datasets/{repo}/resolve/main/{path}` URL pattern. This bypasses the HF API rate limit.

```python
import requests

# Define the dataset repository and file path
repo = "your-dataset-repo"
file_path = "your-file-path"

# Construct the HF CDN URL
cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{file_path}"

# Download the dataset file from the HF CDN
response = requests.get(cdn_url)
with open(file_path, "wb") as f:
    f.write(response.content)
```

#### Step 2: Optimize Dataset Loading using PyArrow
Use PyArrow to load the downloaded dataset files. Avoid using `load_dataset(streaming=True)` for repositories with heterogeneous file schemas.

```python
import pyarrow.parquet as pq

# Load the dataset file using PyArrow
table = pq.read_table(file_path)

# Project the table to {prompt, response} only
prompt_response_table = table.select(["prompt", "response"])
```

#### Step 3: Implement Lightning Studio Reuse
Reuse existing Lightning Studios to avoid recreating them. This saves 80hr/mo quota when iterating training scripts.

```python
import lightning

# Define the Teamspace and studio name
teamspace = "your-teamspace"
studio_name = "your-studio-name"

# List existing studios in the Teamspace
studios = teamspace.studios

# Reuse an existing studio if it exists
for studio in studios:
    if studio.name == studio_name and studio.status == "Running":
        # Use the existing studio
        studio.use()
        break
else:
    # Create a new studio if none exists
    studio = teamspace.create_studio(studio_name)
```
