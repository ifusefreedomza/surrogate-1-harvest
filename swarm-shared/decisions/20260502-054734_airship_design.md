### Highest-Value Incremental Improvement
Implementing HF CDN Bypass for Faster Data Ingestion
#### Problem Statement
The current data ingestion pipeline in Airship is limited by Hugging Face API rate limits, causing delays in training and model updates.

#### Solution
Apply the HF CDN Bypass pattern to download dataset files directly from the CDN, bypassing API rate limits.

#### Implementation Plan
1. **Identify dataset repositories**: Determine which Hugging Face dataset repositories are used in the Airship project.
2. **Get dataset file paths**: Use the Hugging Face API to retrieve a list of file paths for each dataset repository. This can be done using the `list_repo_tree` method with `recursive=False`.
3. **Save file paths to JSON**: Store the file paths in a JSON file for later use in the training script.
4. **Modify training script**: Update the training script to download dataset files directly from the CDN using the saved file paths.
5. **Use CDN download URL**: Construct the CDN download URL for each file using the format `https://huggingface.co/datasets/{repo}/resolve/main/{path}`.

#### Code Snippets
```python
import json
import requests

# Get dataset file paths
repo = "dataset/repo"
file_paths = []
response = requests.get(f"https://huggingface.co/api/repos/{repo}/tree/main")
if response.status_code == 200:
    files = response.json()
    for file in files:
        file_paths.append(file["path"])

# Save file paths to JSON
with open("file_paths.json", "w") as f:
    json.dump(file_paths, f)

# Modify training script to download from CDN
with open("file_paths.json", "r") as f:
    file_paths = json.load(f)

for file_path in file_paths:
    cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{file_path}"
    response = requests.get(cdn_url)
    if response.status_code == 200:
        # Save file to local storage
        with open(file_path, "wb") as f:
            f.write(response.content)
```
#### Expected Outcome
By implementing the HF CDN Bypass, Airship can ingest dataset files more efficiently, reducing the impact of Hugging Face API rate limits on training and model updates. This should result in faster iteration and improved overall performance.
