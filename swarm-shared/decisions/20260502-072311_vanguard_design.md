# vanguard / design

* The Vanguard project lacks a comprehensive README file, making it challenging for new developers to understand the project's purpose, context, and functionality.
* The absence of a README file leads to increased onboarding time and potential errors due to lack of documentation.
* The project's design focus is hindered by the lack of a clear and concise overview of the project's architecture and components.
* The recent commits and swarm-shared decisions indicate a need for improved documentation and onboarding process.
* The project's complexity and multiple patterns and fixes suggest a need for a clear and organized documentation system.

### 2. **Proposed change**
Create a comprehensive README file in the project's root directory (`/opt/axentx/vanguard/README.md`) that includes:
* Project overview and purpose
* Architecture and components
* Onboarding instructions and setup guide
* Links to relevant documentation and resources
* List of patterns and fixes with brief descriptions and tags
