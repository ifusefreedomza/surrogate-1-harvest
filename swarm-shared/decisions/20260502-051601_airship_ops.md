### High-Value Incremental Improvement: Implementing HF CDN Bypass for Faster Data Ingestion
#### Problem Statement
The current data ingestion pipeline in Airship is limited by Hugging Face API rate limits, causing delays in training and model updates.

#### Solution Overview
To address this issue, we will implement the HF CDN Bypass pattern, which allows us to download public dataset files directly from the Hugging Face CDN without hitting the API rate limit. This will significantly speed up our data ingestion pipeline.

#### Implementation Plan

1. **Update `train.py` script**:
	* Pre-list file paths once using `list_repo_tree(path, recursive=False)` and save the list to a JSON file.
	* Embed the JSON file in the `train.py` script to fetch files directly from the CDN.
2. **Modify `dataset-mirror` script**:
	* Project dataset files to `{prompt, response}` only before uploading to the CDN.
	* Move attribution to the filename pattern (`batches/mirror-merged/{date}/{slug}.parquet`).
	* Remove `source` and `ts` columns from the dataset files.
3. **Update `docker-compose.microservices.yml`**:
	* Add a new service for the HF CDN Bypass script.
	* Configure the service to run the script periodically to fetch new dataset files.
