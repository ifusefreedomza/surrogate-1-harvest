### High-Value Incremental Improvement for Workio Ops
#### Diagnosis
The Workio project requires enhancements in its operations to improve the overall system's functionality, efficiency, and user experience. Based on the patterns and lessons learned, the highest-value incremental improvement that can be shipped in <2h is to implement a script to automate the process of restarting Lightning Studio when it stops due to idle timeout.

#### Implementation Plan
1. **Create a new script**: Create a new Bash script (e.g., `restart_lightning_studio.sh`) in the `/opt/axentx/workio` directory.
2. **Check studio status**: Use the Lightning API to check the status of the studio before each `.run()` call.
3. **Restart studio if stopped**: If the studio is stopped, restart it using the `target.start(machine=Machine.L40S)` method.
4. **Schedule the script**: Schedule the script to run every 5 minutes using `cron` to ensure the studio is restarted if it stops due to idle timeout.

#### Code Snippet
```bash
#!/usr/bin/env bash

# Check studio status
studio_status=$(lightning studio status)

# Restart studio if stopped
if [ "$studio_status" == "Stopped" ]; then
  lightning studio start --machine L40S
fi
```
#### Example Cron Job
```bash
*/5 * * * * bash /opt/axentx/workio/restart_lightning_studio.sh
```
